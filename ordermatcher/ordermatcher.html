Order Matcher Application

The following is an order-matching application. Not a particularly mathematically interesting project, but I did get the opportunity to work with a few neat technologies. I wrote the application in C#/Mono with the Nancy framework, and MongoDB for the database, and had never touched C# prior to about six days ago. Not bad for a week's work, I would say.

The application will take orders in the ‘Add Order’ tab, where you enter in the name of the security, select a side (buy or sell), a quantity, and a price. This information is then used to create a new order, which is stored in the order database. As it's being stored in the database, the server checks to see if there are orders of the opposite side with a suitable price (a lower sell price if the added order is buy and higher buy price if sell) that have not been fulfilled yet. These are then used to create transactions which keep track of the IDs for the buy and sell orders and their prices. You can check out the full list of orders in the ‘View Orders’ tab and the list of transactions in the ‘View Transaction’ tab.

Check out my GitHub for the source code.
